# Review Workflow

**目標**

用固定節奏清掉日更帶來的噪音，保持 repo 可查、可用、可維護。

**每週 review**

1. 清理 `raw/inbox/`
2. 檢查是否有 article card 尚未 merge 回 concept pages
3. 合併重複或過薄的 concept pages
4. 檢查是否需要新建或重寫 `wiki/maps/`
5. 精簡 `index.md`，只保留穩定入口
6. 在 `log.md` 記錄 review / cleanup

**每月 review**

1. 找出高頻主題與高頻作者
2. 檢查 concept taxonomy 是否仍合理
3. 補齊缺少的 map page
4. 清查已失去價值的 stub pages
5. 列出值得做 deeper synthesis 的主題

**健康檢查問題**

- 哪些 article card 沒有被任何 concept page 吸收
- 哪些 concept page 其實只有一篇來源支撐
- 哪些 map page 已經失效或太薄
- 哪些 topic 應拆分成更細的概念
